# Проект my_prj
## Описание:
Данный проект создан для выполнения домашнего задания по программе обучения "Python-разработчик".
На данном этапе проходит знакомство с возможностями и работай в GitHab. Функции, представленные на данном этапе
разработки, позволяют:
- обрабатывать информацию о банковских картах и счетах, выводя замаскированный номер и наименование;
- обрабатывать даты и выводить их в нужном формате;
- обрабатывать списки словарей и выводить отсортированный список по датам;
- обрабатывать списки словарей и выводить только те словари, которые подходят по заданному статусу.  
## Установка и использование:
1. Откройте PyCharm
2. Клонируйте репозиторий:
```
git clone https://github.com/svetlana-rogova/my_prj
```
3. Установите зависимости через Poetry:
```
poetri install
```
4. Активируйте виртуальное окружение:
```
poetry shell
```
5. Для маскировки банковских карт и счетов перейлите в модуль widget.py. В print(mask_account_card("   "))
введите наименование и номер карты/счета, нажмите на кнопку Run. 
6. Для вывода даты в нужном формате в этом же модуле введите нужную дату в скобки 
print(get_date("   ")). Нажмите Run.
7. Для работы со словарями необходимо открыть модуль processing.py. При вводе списка словарей в квадратные скобки
print(filter_by_state([])) или print(sort_by_date([])) получим список словарей по параметру 'state', который можно 
выбрать после запятой или отсортированный список словарей по дате.
8. Для получения данных по транзакциям в модуле generators.py можно воспользоваться двумя функциями filter_by_currency()
и transaction_descriptions(), которые позволяют увидеть последовательно транзакции по выбранному параметру при вызове и 
посмотреть описание каждой из них соответственно.
9. Для генерации номера банковской карты в модуле generators.py представлена функция card_number_generator(), которая 
генерирует номера исходя их начального и конечного значений, переданных в параметры.
10. В документе в модуле decorators.py представлен декоратор, который будет автоматически логировать начало и конец 
выполнения функции, а также ее результаты или возникшие ошибки. Для его работы необходимо указать @log() перед вызовом 
функции. В () можно указать конкретный документ, куда будут направляться результаты функции и логи. Если оставить скобки
пустыми, то информация будет выведена в консоль.
11. В модуле reading_transactions.py находяься функции для чтения csv и excel файлов. Для примера использования в 
директории data находятся файлы с вышеуказанными расширениями. 
## Тестирование:
Все тесты лежат в директории tests/, имена модулей начинаются с test_.
В данной директроии реализованы тесты для каждого модуля из пакета src/.
Для запуска тестирования необходимо установить pytest:
```
poetry add --group dev pytest
```
Запустить тесты можно через команду pytest в терминале:
```
pytest
```
Чтобы проверить покрытие файла тестами нужно установить petest-cov в отдельную группу:
```
poetry add --group dev pytest-cov
```
Проверить покрытие можно через команду:
```
poetry run pytest --cov
```
